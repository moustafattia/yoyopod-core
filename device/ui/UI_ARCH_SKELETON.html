YoyoPod UI — Architecture Skeleton A buildable design that combines the web-framework structure lens with the game-engine aesthetics lens. This is the blueprint: layers, folder tree, struct shapes, enum definitions, scene construction, deck contracts, and the hard limits the SPI panel imposes on what the system can do per frame.

The first three reports were descriptive. This one is prescriptive. It is the full architectural skeleton — every layer named, every responsibility assigned, every key struct and enum sketched in Rust. You should be able to read this file and start filing the folder tree on disk; nothing here is hand-wavy.

1. The unifying thesis

Two lenses, one architecture:

Web-framework lens

Structure. Composition, reconciliation, declarative screens. Answers: how do I build a screen out of reusable parts?

Game-engine lens

Aesthetics. Layers, particles, timelines. Answers: how do I make a screen feel alive?

They map onto orthogonal parts of the architecture and do not compete:

ConcernOwnerLens
How a scene is composed from componentsscene + componentsWeb
How the tree is diffed against the previous frameengine::reconcilerWeb
How a scene is split into z-ordered layersscene::layersGame
How motion is described as dataanimation (timelines)Game
How "look alive" effects are producedscene::fxGame
How user input becomes navigationrouterWeb
How content is laid out as repeated itemsscene::deckBoth
The unification. A Scene is both a composition (web) and a layered, animated stage (game). The same data structure carries both. We build it with component functions (web) and render it with z-ordered layer composition + timelines (game).

3. Hardware constraints (the budget)

Every design decision below is shaped by these. Optimism here is paid for with dropped frames.

Display
240 × 280, RGB565
Whisplay panel. ~134 KB per full frame.
SPI bus
~40 MHz typical
~27 ms theoretical floor for a full-frame flush, plus overhead. ~15–20 fps is the hard ceiling for full-frame redraws.
Region flush
status bar = 240×32
~3.1 ms theoretical, ~5–8 ms real. Status bar can update at 30+ fps without touching the rest.
Frame budget
~50 ms target
20 fps target → 50 ms per frame for reconcile + LVGL + SPI. Engine work must fit in <5 ms.
LVGL widget count
≤ 60 per scene
Each widget is a heap allocation + style state. Above this, LVGL layout passes get slow.
Active timelines
≤ 16
Each timeline is sampled per frame; keep linear scans cheap.
Particle count
≤ 8 visible
Each particle is one LVGL widget. Recycle aggressively.
Memory
tight
Embedded MCU class. Avoid Box<dyn Trait> in hot paths; prefer enums and stack values.

What this forces

  • Two render rates. A "slow" rate (≤20 fps) for full-frame; a "fast" rate (≤30 fps) for the status-bar region. The engine separates them.
  • Reconciliation must be cheap. The diff algorithm is keyed and positional only — no fancy heuristics, no string-matching.
  • No closures in the tree. Owned data only.
  • No allocator-on-every-frame. Reuse buffers (mutations, timeline samples) across frames.
  • Looping animations should not force redraws. A breathing halo redraws only when its sampled opacity actually changed enough to matter (quantize to 8 levels).

5. Core types (the language of the engine)

These are the types that flow between layers. Everything else is built from them.

Identity

#[derive(Debug, Clone, PartialEq, Eq, Hash)]
pub enum Key {
    Static(&'static str),     // "hub_card_panel"
    Indexed(usize),             // for rows in a deck
    String(String),             // dynamic id from snapshot (contact.id, track.id)
}

6. Scenes (construction & lifecycle)

A Scene is the unified type: it's both a component composition (web) and a layered render description (game).

// scene/scene.rs

pub struct Scene {
    pub id:         SceneId,            // UiScreen + a generation counter
    pub backdrop:   Backdrop,
    pub stage:      Stage,
    pub decks:      Vec<Deck>,
    pub cursor:     Option<Cursor>,
    pub fx:         FxLayer,
    pub modal:      Option<Modal>,
    pub timelines:  Vec<Timeline>,      // scene-local timelines (SceneTime clock)
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub struct SceneId {
    pub screen:     UiScreen,
    pub generation: u32,                // for invalidation if route params change
}

Note what the Scene type does not contain:

  • No hud — the HUD is a separate persistent scene (see SceneGraph below).
  • No component state — scenes are pure data, regenerated from props each render.
  • No widget handles — those live in the renderer's NodeRegistry.

SceneGraph (lifecycle owner)

// scene/graph.rs

pub struct SceneGraph {
    pub hud:           HudScene,                   // persistent — survives navigations
    pub active:        Scene,                      // currently visible world scene
    pub history:       Vec<ScenePushFrame>,        // for back navigation
    pub modal_stack:   Vec<Modal>,                 // drawn on top of active
    pub global_clock:  GlobalClock,                // ticks at GlobalTime
}

pub struct ScenePushFrame {
    pub route:       UiScreen,
    pub params:      RouteParams,
    pub cached_state: SceneCacheEntry,             // for KeepAlive scenes
}

pub enum SceneCacheEntry {
    Discarded,                                     // recompute on return
    Retained { actor_state: ActorState },          // bring back without re-init
}

Component contract (how a screen builds its scene)

// components/screens/hub.rs

pub struct HubProps {
    pub cards:          Vec<HubCardModel>,
    pub selected_index: usize,
    pub chrome:         ChromeProps,
}

pub fn props_from(snapshot: &RuntimeSnapshot, focus: usize) -> HubProps { ... }

pub fn scene(props: &HubProps) -> Scene {
    Scene {
        id:        SceneId::new(UiScreen::Hub),
        backdrop:  Backdrop::AccentDrift { accent: focused_accent(props), speed_ms: 800 },
        stage:     Stage::CenteredHeroIcon,
        decks:     vec![ hub_deck(props) ],
        cursor:    Some(Cursor::UnderlineDots {
                       count: props.cards.len(),
                       focus: props.selected_index,
                   }),
        fx:        FxLayer {
                       halos: vec![ presets::breathe_around(RegionId::HeroIcon) ],
                       particles: vec![ presets::backdrop_drift(focused_accent(props)) ],
                       ..Default::default()
                   },
        modal:     None,
        timelines: vec![ presets::scene_enter() ],
    }
}

That's the entire contract: props_from(snapshot, focus) → Props, then scene(&props) → Scene. No state, no widget handles, no LVGL.

7. Layer slots (the z-stack)

Every Scene has exactly 7 z-ordered slots. The engine flattens them into a single Element tree in z-order for the reconciler. Layers below in the table are drawn behind layers above.

z=0 Backdrop Solid color, gradient, or accent drift. One enum: Backdrop.
z=1 Stage Layout template (fixed geometry). Defines named regions (HeroIcon, ListBody, etc.) that decks slot into.
z=2 Decks Vec of Deck. Data-driven content (cards, rows, pages).
z=3 Cursor Optional. The selection actor that tweens between deck positions.
z=4 FxLayer Halos, pulse rings, particle fields. Driven by timelines.
z=5 HUD Persistent across scenes. Status bar + footer. Owned by SceneGraph, not Scene.
z=6 Modal Optional pop-ups. Loading, Error, future dialogs. Owned by SceneGraph::modal_stack.

Backdrop

// scene/backdrop.rs

#[derive(Debug, Clone, Copy)]
pub enum Backdrop {
    Solid(u32),                                          // base color
    Gradient { from: u32, to: u32, angle_deg: i16 },
    AccentDrift { accent: u32, speed_ms: u32 },          // slow tween toward accent
    Vignette { base: u32, falloff: u8 },
}

8. Decks (the content layer)

A deck is the single abstraction for "repeated items in a region". Every list, card row, and pager in the worker is a deck.

// scene/deck.rs

pub struct Deck {
    pub kind:           DeckKind,
    pub region:         RegionId,            // where on stage
    pub items:          Vec<DeckItem>,
    pub focus_index:    usize,
    pub focus_policy:   FocusPolicy,         // None | Wrap | Clamp
    pub item_anim:      DeckItemAnim,
    pub swap_anim:      Option<Transition>,  // when items change
    pub recycle_window: Option<usize>,       // virtualize beyond N
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum DeckKind {
    CardRow,     // Hub: 4 horizontal cards, one visible at a time
    List,        // Listen/Talk/Contacts: vertical rows
    Page,        // Power: full-screen pages, advance to cycle
    Grid,        // future: e.g. 2x2 quick-actions
    Buttons,     // VoiceNote/TalkContact: row of action buttons
}

#[derive(Debug, Clone)]
pub struct DeckItem {
    pub key:      Key,                       // stable identity (contact.id, etc.)
    pub render:   ItemRender,                // how to draw this item
}

#[derive(Debug, Clone)]
pub enum ItemRender {
    Card    (CardModel),
    Row     (RowModel),
    Page    (PageModel),
    Button  (ButtonModel),
}

#[derive(Debug, Clone, Copy)]
pub enum DeckItemAnim {
    None,
    ScaleOnFocus { from_permille: u16, to_permille: u16 },
    BreatheWhenFocused,
    StaggerEnter { delay_per_index_ms: u32 },
}

How a deck becomes Elements

Each DeckKind has a deterministic layout: CardRow shows the focused item full-size with side-peeks; List stacks rows vertically clipped to the region; Page shows one item at a time; Buttons arranges items horizontally. The deck-to-element conversion lives in scene/deck.rs and is shared across all screens.

9. Components (composition vocabulary)

Three tiers, with strict rules about who calls whom:

TierReturnsCan callExamples
PrimitivesElementNothing — leaf builders.label(), container(), image(), progress()
WidgetsElement subtreePrimitives, other widgets.status_bar(), list_row(), card(), icon_halo(), chip()
ScreensSceneWidgets, primitives, deck builders.hub::scene(), now_playing::scene()

Primitive (example)

// components/primitives/label.rs

pub fn label(role: &'static str) -> Element {
    Element {
        key: None,
        kind: ElementKind::Label,
        role: Some(role),
        props: ElementProps::default(),
        layout: Layout::Region(RegionId::Auto),
        anim: None,
        children: vec![],
    }
}

impl Element {
    pub fn text(mut self, t: impl Into<String>) -> Self { self.props.text = Some(t.into()); self }
    pub fn icon(mut self, k: impl Into<String>) -> Self { self.props.icon_key = Some(k.into()); self }
    pub fn accent(mut self, rgb: u32) -> Self { self.props.accent = Some(rgb); self }
    pub fn selected(mut self, b: bool) -> Self { self.props.selected = Some(b); self }
    pub fn child(mut self, e: Element) -> Self { self.children.push(e); self }
    pub fn key(mut self, k: impl Into<Key>) -> Self { self.key = Some(k.into()); self }
}

Widget (example)

// components/widgets/list_row.rs

pub fn list_row(row: &ListRowModel, accent: u32) -> Element {
    container("list_row")

Screen (example, abridged)

// components/screens/listen.rs

pub fn scene(props: &ListenProps) -> Scene {
    Scene {
        id: SceneId::new(UiScreen::Listen),
        backdrop: Backdrop::Solid(theme::BACKGROUND_RGB),
        stage: Stage::ListWithChrome,
        decks: vec![
            Deck {
                kind: DeckKind::List,
                region: RegionId::ListBody,
                items: props.rows.iter()

                        render: ItemRender::Row(r.into()),

                focus_index: props.focus,
                focus_policy: FocusPolicy::Wrap,
                item_anim: DeckItemAnim::StaggerEnter { delay_per_index_ms: 40 },
                swap_anim: None,
                recycle_window: Some(4),
            },
        ],
        cursor: Some(Cursor::RowGlow),
        fx: FxLayer::default(),
        modal: None,
        timelines: vec![presets::scene_enter()],
    }
}

10. Stages (layout templates)

A Stage is a named layout template — the fixed geometry that backdrop sits on and decks slot into. It defines a small set of named regions.

// scene/stage.rs

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum Stage {
    CenteredHeroIcon,    // Hub, Ask, VoiceNote-ready — big icon + title + subtitle + dots
    ListWithChrome,      // Listen, Talk, Contacts, CallHistory, Playlists, Recents
    NowPlayingPanel,     // NowPlaying — icon halo + title + artist + progress
    CallPanel,           // Incoming, Outgoing, InCall
    TalkActionsGrid,     // TalkContact, VoiceNote-review — header + button row
    PagedDetail,         // Power — multi-page detail
    OverlayCenter,       // Loading, Error
}

#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub enum RegionId {
    Auto,           // let the stage decide
    Backdrop,
    Header,
    HeroIcon,
    ListBody,
    ButtonRow,
    Dots,
    Progress,
    Title,
    Subtitle,
    Footer,

Why three clocks. Looping FX (breathing halo) needs to keep going across scene swaps — that's GlobalTime. Scene-intro stagger needs to reset when the scene mounts — that's SceneTime. One-shot ripples need to start when an event fires (not when the scene mounted) — that's EventTime. Same primitive, three different "when does t=0 happen?" answers.

14. Engine (reconciler + scheduler)

// engine/mod.rs

pub struct Engine {
    pub tree_cache:  TreeCache,      // last Scene → Element tree
    pub node_alloc:  NodeIdAlloc,    // monotonic NodeId counter
    pub mutations:   Vec<Mutation>,  // reused buffer; cleared per frame
    pub timelines:   ActiveTimelines,
}

impl Engine {
    pub fn render(&mut self, graph: &SceneGraph, now_ms: u64) -> &[Mutation] {
        self.mutations.clear();
        let new_tree = flatten(graph);                       // 7 layers → one Element tree
        self.reconciler.diff(
            self.tree_cache.previous(),
            &new_tree,
            &self.timelines.sampler(now_ms),
            &mut self.mutations,
        );
        self.tree_cache.replace(new_tree);
        &self.mutations
    }

    pub fn schedule_timeline(&mut self, t: Timeline) { ... }
    pub fn tick_clocks(&mut self, now_ms: u64) { ... }
}

Reconciler algorithm (sketch)

  1. Walk old and new trees in parallel.
  2. At each position: same key + same kind → reuse NodeId, diff props, emit Updates.
  3. Different key or kind → emit Remove for old subtree, Create+recurse for new.
  4. If parent's child order changed → emit Reorder.
  5. Layout: each Element's Region resolves to a rect via region_rect(stage, region); emit Place if it changed.
  6. For each animating Element (has AnimSlot), sample the timeline and emit a Update with the interpolated value.

Scheduler — what does it do per tick?

fn tick(graph: &mut SceneGraph, engine: &mut Engine, now_ms: u64) -> FrameOutcome {
    // 1. Advance clocks (GlobalTime always; SceneTime if scene is active)
    engine.tick_clocks(now_ms);

    // 2. Decide frame rate. If only the status bar dirty bit is set → fast path.
    let mode = if hud_only_dirty(graph) { RenderMode::HudRegion } else { RenderMode::FullFrame };

    // 3. Reconcile.
    let mutations = engine.render(graph, now_ms);

    // 4. Hand to renderer.
    FrameOutcome { mode, mutations }
}

17. Data flow per frame

Putting it all together — what happens between a button press and a pixel change:

1. button → transport::handle_button_input
2. transport::dispatcher → application::UiRuntime::handle_input(action)
3. UiRuntime delegates to router (select / back / passthrough policies)
4. Router updates active_screen + screen_stack; may emit UiIntent
5. UiIntent → outbound::emit_event → manager (over JSON)
6. UiRuntime marks DirtyState (focus + input bits)
7. main loop calls UiRuntime::render_if_dirty
       │
       ▼
8. props_from(snapshot, focus) for the active route
9. components::screens::<active>::scene(&props)              → Scene
10. SceneGraph.update(active = scene, modal = …, hud = unchanged)
11. engine.tick_clocks(now_ms)
12. engine.render(&graph, now_ms)                            → &[Mutation]
13. renderer.apply(&mutations)                               → LVGL ops
14. renderer.flush(fb, mode)
       │
       ▼
15. hardware::flush_region(fb, region) over SPI
16. UiRuntime.mark_clean()

Summary

Four layers (application → engine → renderer → platform). Seven scene layer slots (backdrop → stage → decks → cursor → fx → hud → modal). Three component tiers (primitives → widgets → screens). Three animation clocks (SceneTime / GlobalTime / EventTime). One hardware budget (~20 fps full-frame, ~30 fps HUD region, ≤60 widgets per scene).

Components return Scenes; the engine reconciles them into Mutations; the renderer plays Mutations into LVGL widgets; the platform flushes pixels over SPI. Web and game lenses unify at the Scene type: it's a composition (web) and a layered animated stage (game), built by a pure function from props.

The folder tree maps one-to-one onto the architecture. The struct sketches in this report compile (modulo a handful of small types I omitted for brevity). Build it stage by stage following the roadmap from the previous two reports; each stage delivers value independently.
